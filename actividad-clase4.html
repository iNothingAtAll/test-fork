🪜 2. REST o Eventos

Interacción REST Evento ¿Por qué?
1 x Un servicio emisor publica eventos sin conocer a los destinatarios y los servicios consumidores reaccionan cuando los reciben
2 x se require un servicio concreto para recibir una respuesta directa al finalizar las autenticaciones
3 x Se espera una respuesta inmediata despues de efectuar la verificaciones dentro de la base de datos
4 x Se hace uso de servicios externos o de terceror lo que facilita su implementacion

🪜 3. Síncrono o Asíncrono

Interacción Síncrono Asíncrono Justificación
1 x Se espera que el sistema de apuestas funcione de forma asincronica, el uso debe ser simultaneo sin importar la cantidad de peticiones
2 x Este debe sincrono, ya que debe tener una sere de procesos antes de efectuar una accion de pago
3 x Este puede ser sincrono ya que necesita verificaciones en las base de datos para efectuar una autenticacion de usuario
4 x Este puede ser sincrono ya que debe verificar el uso saldos para hacaer las recargas o retiros
